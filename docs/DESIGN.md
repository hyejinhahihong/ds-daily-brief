## 2. 제목 언어 — content_type별 분기 (v3 신규)

**문제:** v2는 영문 원문 제목 16개 + 한글 본문. 눈이 제목마다 읽기 모드를 전환한다.
**해결:** Item에 `title_ko` 필드 추가(SPEC §8). content_type으로 "주 제목/부제목"을 가른다.

| content_type | 주 제목 (크게) | 부제목 (작게·회색) | 근거 |
|---|---|---|---|
| **📄 paper** | **원문 제목 유지** | `title_ko`(한글 한 줄 요약) | 논문 제목은 학술 검색 키. 번역하면 원문을 못 찾는다. |
| **📰 news / ✍️ blog / 📦 release** | **`title_ko`**(한글 재작성) | 원문 제목 | 정보 전달이 목적 → 한글이 낫다. |

**`title_ko` 작성 규칙 (write.py):**
- 직역 금지. **정보를 담은 재작성.** 수치·고유명사·결론을 제목에 넣는다.
- 예: `Google, TabFM 공개 — 학습 없이 표 데이터 분류·회귀, TabArena 51개 데이터셋 검증`
- 40~60자. 고유명사(TabFM, XGBoost, NeurIPS)는 **원문 표기 유지.**
- 부제목은 `--text-2`, 작게. 주 제목과 시각 위계가 분명해야 한다.

## 3. 아이템 구조 = 아코디언 + bold 강조 (v2 아코디언 유지 + 강조 규칙 신규)

**네이티브 `<details>/<summary>`.** JS 불필요, 접근성 무료, 인앱 브라우저 호환.

### 정렬: 거터 + 단일 콘텐츠 컬럼 (v4 — 정렬선 4개 → 1개)
- **CSS Grid `grid-template-columns: 40px 1fr`.** 번호(01/02)만 거터, 나머지 **전부 콘텐츠 컬럼**.
- 콘텐츠 컬럼 안 모든 요소(제목/부제/메타/요약/WHY/WHAT)는 **같은 x에 정렬.** 들여쓰기·행잉 인덴트 금지.
- **배지는 제목 앞에서 빼서 메타 라인으로 내린다.** 배지는 라벨이 아니라 메타데이터 → 출처와 같은 급, 같은 줄.
  구분자 `·`. 스캔 앵커는 번호(01/02)가 이미 한다.
- 모바일(<900px) 거터 40px → **28px**로 축소.

**항상 표시 (`<summary>`, 클릭 시 펼침):**
```
01 │ 주 제목 (한글 또는 원문, content_type별)       ← 제목은 --text. 링크 아님(펼치기용)
   │ 부제목 (title_ko 또는 원문)                     ← --text-2, 작게
   │ ✍️ 블로그 · developer.nvidia.com · KDD 2026     ← 배지+출처+venue, 전부 메타 (--text-2)
   │ 요약 첫 문장…                                    ← 펼칠지 판단 재료 (--text-2)
```

**접힘 (`<details>` 본문):**
```
나머지 요약 (첫 문장 이후)
WHY IT MATTERS  — 왜 중요한가 1~2문장
WHAT'S DIFFERENT — 비교 대상 명시 1~2문장 (없으면 슬롯 통째 생략, SPEC 원칙 6)
↳ 연결된 이전 기사 (있을 때만)
#태그 …
[원문 보기 →]                                      ← --accent CTA
```

### bold 강조 — 형광펜 금지 (v3 신규)
- summary / why_it_matters / whats_different 안에서 **핵심 구절만 `<strong>`.**
- **규칙: 문단당 최대 2개.** 없으면 안 넣어도 된다(0개 허용).
- 대상: 수치(`SWE-bench 59.0%`), 결론(`XGBoost를 능가`), 핵심 고유명사.
- **`<strong>`은 굵기 600, 색 `--text` 유지.** 액센트 색·배경색 절대 안 씀.
- **형광펜(배경 하이라이트) 금지.** 16건 × 형광펜 = 다시 시끄러워진다. 배지 색을 폐기한 것과 같은 원칙. **bold가 이미 하이라이트다.**
- 구현: write.py가 `**...**`(마크다운 bold) 마커로 출력 → render가 **먼저 escape 후** 마커만 `<strong>`으로 치환(인젝션 안전).

- **배지**: `📄 논문` `📰 뉴스` `📦 릴리스` `✍️ 블로그`. **메타 라인에 위치**(제목 앞 아님, v4). 색 없음 — 이모지가 구분자.
- **넘버링 `01/02/03`**: 카테고리별, **거터**에 배치. 접힌 리스트의 스캔 앵커 겸 유일한 좌측 기준선.
- **TOP 3 아이템은 기본 펼침**(`<details open>`). 나머지 접힘.

## 5. 레이아웃 + 목차 (v3 개정 — 데스크톱 중앙 정렬 + 3단 브레이크포인트)

### 데스크톱 쏠림 해결 (v3 신규)
v2는 [좌측 TOC + 본문] 덩어리가 왼쪽에 붙고 우측 여백이 컸다(큰 모니터에서 목이 돌아간다).
- **[TOC + 본문] 덩어리 전체를 뷰포트 중앙 정렬.** max-width 컨테이너로 감싸고 `margin: 0 auto`.
- 본문 폭 **680~720px 그대로 유지.** 넓히지 말 것.

### 브레이크포인트 3단 (v3 재정의)
| 폭 | 레이아웃 | 근거 |
|---|---|---|
| **< 900px** | 단일 컬럼. **상단 목차 기본 펼침.** | 카톡 인앱 브라우저 = 주 뷰포트 |
| **900 ~ 1199px** | **단일 컬럼 유지. 사이드 TOC 숨김 → 상단 목차로 폴백.** | TOC 260 + 본문 720 = 980px이라 이 구간에서 서로 밀어낸다. 창을 반만 연 노트북. |
| **≥ 1200px** | 좌측 sticky TOC(260px) + 본문(720px), **덩어리 중앙 정렬.** | 큰 모니터 |

- **판단·근거**: 900~1199 구간은 사이드 TOC를 **숨기고 상단 목차로 폴백**한다. 260+720=980에 gap·padding까지
  더하면 1199px에서도 사이드바와 본문이 서로 밀려 본문이 720 밑으로 찌그러진다. 상단 목차는 이미 모바일에서
  검증된 폴백이라 재사용이 안전하다. 사이드 TOC는 여유가 확실한 1200px부터만 켠다.
- 상단 목차와 사이드 TOC는 **같은 `nav.toc` 하나**를 CSS로 위치만 바꾼다(중복 DOM 없음).

### 목차 — "…" 잘림 해결 (v3 개정)
- **1위 아이템 제목을 2줄까지 허용**(`-webkit-line-clamp: 2`). **1줄 강제 잘림 금지.**
- `title_ko`가 있으면 `title_ko`를 쓴다(한글이 폭 대비 정보량이 높다).
- **TOC 폭 220px → 260px.**
- 데스크톱(≥1200): 좌측 sticky. 현재 스크롤 위치 카테고리를 `--accent`로 활성 표시(IntersectionObserver).
- TOC에 `id="toc"`(섹션 끝 "목차로 돌아가기" 앵커 대상).

### 아코디언 상호작용 (유지)
- `<details>` 네이티브. TOP 3만 `open`. 섹션 헤더 "전체 펼치기/접기" 토글.
- 아코디언 마커는 회색 ▸/▾. 액센트 안 씀.

- 페이지 **하단**에 8개 카테고리 + 설명 1줄 + `/category/*.html` 링크(불릿 리스트).
- 카테고리 설명은 **위 섹션 헤더와 동일 문구 재사용**(단일 상수).
- **index는 오늘자 그대로 유지**(랜딩으로 바꾸지 않는다). 근거: 카톡 링크로 바로 진입한다. 랜딩을 거치면 클릭이 하나 는다. (DECISIONS.md)
- `/category/*.html`은 **아직 실제 생성 안 함** — 링크만 걸어두고 다음 단계.

---

## 7. 다크 모드 (유지)
- `@media (prefers-color-scheme:dark)` 기본 + 루트 `data-theme` 토글이 항상 이김.
- §1 토큰 표 Dark 열 사용. 액센트 teal은 다크에서 밝은 `#6FC7B5`.

## 8. 하지 않는 것
- ❌ 제목·배지·bold에 액센트/컬러
- ❌ **형광펜 / 배경 하이라이트** (16건 × 형광펜 = 다시 시끄러움)
- ❌ 카테고리 8색 / 컬러 코딩
- ❌ 히어로 이미지 / 카드 그리드 / CTA 버튼 중심 레이아웃
- ❌ 전문 16개 동시 펼침 (스캔 불가 → 아코디언)
- ❌ 넓은 데스크톱 풀블리드 / 좌측 쏠림
- ❌ index를 랜딩 페이지로 전환
- ❌ `/category/*.html` 실제 생성 (링크만)
